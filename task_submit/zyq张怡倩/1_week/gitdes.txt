git工作流程：git会自动记录做了哪些修改，并且可以多人同时把别人的代码复制到本地，做了修改后再传回，这样可以有多个版本，容易找回之前的版本，也使修改一目了然。

git clone                                  把远程库复制到本地库

cd                                          进入文件

git branch                                  查看当前分支

git checkout                               进入自己的分支

git status                                   查看当前文件状态

git add .                                     添加文件

git commit -m “description”                   添加修改描述

git push                                      传到远程分支

Git的分支原理：分支就像平行时空，每个分支互不干扰。每个人先各自在自己的分支上进行任务，完成后再把自己的分支上传合并到原来分支

Git分支合并：git merge
